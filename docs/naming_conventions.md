# Naming Conventions

Use lowercase snake_case file names that describe both the content type and concept.

## Term Entry Examples

- `disease_thyroid_cancer.example.json`
- `biomarker_thyroglobulin.example.json`
- `tissue_thyroid.example.json`
- `method_rna_seq.example.json`

## Query Fragment Examples

- `pubmed_thyroid_cancer_basic.example.json`
- `geo_thyroid_cancer_basic.example.json`

## General Rules

- Prefix term files with the category when possible.
- Use stable English identifiers in filenames and IDs.
- Keep filenames descriptive but short.
- Reserve `.example.json` for public sample files in this repository.
