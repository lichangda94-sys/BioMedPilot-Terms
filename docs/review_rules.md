# Review Rules

Public contributions are reviewed as candidate resources rather than automatic production assets.

## Review Principles

- schema validity is required
- duplicate checks should be performed
- source appropriateness matters
- no patient or private data is allowed
- no proprietary database copying is allowed
- no overbroad synonyms should be accepted without caution
- database-specific terms require review
- accepted public examples do not imply production integration

## Reviewer Focus

- confirm the JSON file validates against the correct schema
- check whether the concept already exists under another normalized name
- confirm that the wording is appropriate for biomedical use
- review ambiguous synonyms and abbreviations carefully
- verify that no confidential, restricted, or copied commercial material was submitted
