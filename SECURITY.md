# Security Policy

## Sensitive Data Rules

Do not submit any of the following in issues, pull requests, commits, or attachments:

- Secrets or credentials
- API credentials or Tokens
- user data
- patient data
- private institutional data
- private vocabulary assets
- private prompt assets
- query logs
- private database exports
- commercial-only search assets

This repository is intentionally public and should contain only schemas, examples, validation logic, and public documentation.

## Reporting

Report security issues or sensitive-data exposure privately to the maintainer through a non-public channel.

Public issues must not include private data, Secrets, credentials, patient material, or other sensitive records.
